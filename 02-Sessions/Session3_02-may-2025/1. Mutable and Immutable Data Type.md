## ðŸ§  **Immutable Data Types**

These **cannot** be changed after creation. Any modification results in a **new object**.

| Data Type   | Example                       |
| ----------- | ----------------------------- |
| `int`       | `x = 42`                      |
| `float`     | `pi = 3.14`                   |
| `bool`      | `flag = True`                 |
| `str`       | `name = "Python"`             |
| `tuple`     | `coords = (10, 20)`           |
| `frozenset` | `fset = frozenset([1, 2, 3])` |
| `bytes`     | `data = b"ABC"`               |
| `NoneType`  | `x = None`                    |

---

## ðŸ”§ **Mutable Data Types**

These **can** be changed in-place without changing their identity.

| Data Type        | Example                      |
| ---------------- | ---------------------------- |
| `list`           | `my_list = [1, 2, 3]`        |
| `dict`           | `my_dict = {"a": 1, "b": 2}` |
| `set`            | `my_set = {1, 2, 3}`         |
| `bytearray`      | `barr = bytearray(b"ABC")`   |
| `Custom classes` | `class Car: pass`            |

> ðŸ”“ **Why mutable?**  
> They allow updates without reallocating memoryâ€”great for performance but riskier with shared references.
